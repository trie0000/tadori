🎯 Tadori — SharePoint Cookie 認証テスト

コンソールにペーストできない環境向け。下のリンクをお気に入りバーにドラッグするだけで導入できます（コピペ不要）。

導入手順

  1. Edge でお気に入りバーを表示（Ctrl+Shift+B）
  2. 下のボタンをお気に入りバーへドラッグ＆ドロップ
  3. テスト対象の SharePoint サイト（受信メールリストのある画面）を開く
  4. 追加したお気に入り 「📌 Tadori Cookie認証テスト」をクリック
  5. List 表示名を聞かれるので入力（既定: 受信メールリスト）
  6. 画面右上に結果パネルが出ます

📌 Tadori Cookie認証テスト ← これをドラッグ

判定の見方
・READ OK ✅ … Cookie 認証で List が読めた = ADR-007 クリア
・401 / 403 … 条件付きアクセス等で Cookie 認証が阻害 → MSAL フォールバック検討が必要
・取得件数と列名が出るので、追加列（is_ml 等）の有無確認にも使えます

このブックマークレットがやること

現在開いている SharePoint サイトに対し、ブラウザの既存ログイン Cookie を使って /_api/web/lists/getbytitle('...')/items を読み取り、成否・件数・列名を表示します。 書き込み（try-claim）は行いません（読み取り専用の安全な確認）。
